# Images, icons & fonts ‚Äî agent rules

## One pipeline: build time

Images are committed to the repo and transformed at build by **`vite-imagetools`**
(sharp runs once; the output is hashed, immutable and first-party). There is no
runtime resize route and no CDN ‚Äî the blog has no user uploads and no CMS. Do not
add one "just in case".

```ts
import avatar1x from '../images/fabio.jpg?w=72&format=webp&quality=90';
import avatar2x from '../images/fabio.jpg?w=144&format=webp&quality=90';
```

## Two components, and the boundary between them matters

- **`<Picture>`** (`libs/ui`) ‚Äî the primitive. Takes plain `srcset` strings and
  knows nothing about where they came from. Renders `<picture>` + `<img>` with
  intrinsic `width`/`height` (so nothing reflows), lazy + async by default.
- **`<Image>`** (`apps/blog/app/components/image.tsx`) ‚Äî the adapter. It is the
  only place that knows the shape of a `?as=picture` import.

Keep that boundary. It is why swapping the pipeline later would touch one file
instead of every component.

## Rules

- **`alt` is required.** Empty string ONLY for a decorative image whose meaning
  is already in the adjacent text ‚Äî the avatar is the example: it is
  `aria-hidden` with `alt=""`, because the name is written right beside it
  (WCAG 1.1.1).
- **Always pass intrinsic `width`/`height`.** Without them the page reflows when
  the image lands (CLS).
- **Density (`1x`/`2x`) for a fixed-size image** like the avatar; **widths +
  `sizes`** for one that scales with the layout. A `w`-descriptor srcset with no
  `sizes` makes the browser assume `100vw` and fetch the largest variant.
- **`priority` only for the LCP image.** Everything else is lazy.
- **`as=picture` / `as=srcset` must be LAST in the query** ‚Äî `app/images.d.ts`
  matches on the specifier's ending. Anywhere else and the import is typed `any`,
  silently.
- The source file must be **big enough for the 2x variant**. Upscaling is not a
  transform, it is a blur.

## Known gap

There is **no `og:image`**, so every link to the blog previews as bare text. See
[`.agent/seo.md`](./seo.md).

## Icons ‚Äî `libs/icons`

SVGs in `svg/` (vendored from simple-icons), React components generated by SVGR into
`src/` (**git-ignored codegen** ‚Äî never edit it, never commit it).

**Fix the config, never the .svg.** The sources are vendored: the next download undoes
a hand edit. `.svgrrc.cjs` is the guarantee.

Every icon must paint with `currentColor`, so it inherits the surrounding text colour
and follows the accent for free. Two kinds of black have to be handled, and only one is
visible in the file:

- an **explicit** colour (`stroke="#000"`, a brand hex) ‚Üí `convertColors`, which leaves
  `none` alone so a stroke-drawn icon keeps `fill="none"` instead of filling in solid;
- an **implicit** one ‚Äî simple-icons ship a bare `<path d="‚Ä¶">` with no `fill`, and
  SVG's default is black. There is no string to substitute, which is why the old
  `replaceAttrValues: { '#000': ‚Ä¶ }` matched nothing and shipped two black-on-black
  icons. The `iconCurrentColor` plugin gives the root `<svg>` a fill when the file
  declares none.

`tools/check-icons.mjs` fails the build on an icon that hardcodes a colour or never
sets `currentColor` ‚Äî because nobody reviews generated code.

An icon is **decorative**: `aria-hidden`, with the name in an `sr-only` span beside it
(never an `aria-label` on the link ‚Äî it overrides the content, and with it the "opens
in a new tab" hint). `IconLinks` enforces both.

## Fonts

Self-hosted under `apps/blog/public/fonts`, `@font-face` in
`libs/theme/src/styles/fonts.css`. Never load a font from a third-party origin ‚Äî see
[performance](./performance.md) for why (it cost 0.7s of FCP, and made /colophon's
"no tracking" untrue). Metric-adjusted fallbacks are mandatory; regenerate them with
`node tools/font-metrics.mjs`.
